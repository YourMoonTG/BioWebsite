Как поднять бесплатную CDN-систему с Oracle Cloud и GitHub

cdn oracle-cloud github сервер devops инфраструктура

CDN-система транслирует файлы из закрытых репозиториев GitHub через ваш сервер. GitHub ограничивает репозиторий до 15 GB, но позволяет создавать неограниченное количество репозиториев.

Характеристики сервера

До 2 бесплатных серверов в Oracle Cloud:

  • 100 GB SSD
  • 1 GB RAM
  • 2 CPU

Регистрация и создание VPS в Oracle Cloud

Регистрация: https://youtu.be/Hv6mTU-K61s?si=qq03v3Qbk44TOjTy

Создание инстанса

  1. Поиск → "Instance"
  2. "Create instance"
  3. Бесплатный тариф
  4. SSD: 50 GB → 100 GB
  5. OS: Oracle Linux → Ubuntu (мини-версия)
Важно: Сохраните SSH-ключи. Запишите публичный IP адрес.

Подключение по SSH

ssh -i path/to/private_key.pem ubuntu@server_ip

Установка aaPanel

Подготовка системы

sudo apt update -y
sudo apt upgrade -y
sudo apt install git wget -y

Установка aaPanel

wget --no-check-certificate -O install_7.0_en.sh https://www.aapanel.com/script/install_7.0_en.sh
chmod +x install_7.0_en.sh
./install_7.0_en.sh aapanel

Сохраните URL панели, username и password.

Важно: Если IP в URL скрыт, замените на публичный IP сервера. Нужно открыть порты: 30900, 888, 80, 443, 20-21

Настройка портов

После этого нам нужно открыть порты внутри сервера и на сайте Oracle Cloud (открывайте те порты, что написаны в консоли; я буду показывать на примере того, что выдало мне).

Открытие портов на сервере

Для начала давайте откроем порты внутри сервера:

sudo apt install ufw -y
sudo ufw allow 30900
sudo ufw allow 888
sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 20
sudo ufw allow 21

Прописываем: sudo ufw status. Если видим там все эти порты, значит, всё получилось, мы молодцы, и почти всё работает.

Открытие портов в Oracle Cloud

Переходим обратно в панель Oracle Cloud.

В списке наших инстансов выбираете тот, который создавали. В разделе "Сведения об экземпляре" ищите "Виртуальная облачная сеть". В моём случае она называется vcn-20221009-1257.

Далее ищите пункт "Безопасность" и нажимаем на доступный нам список. В моём случае это "Default Security List for vcn-20221009-1257".

После нажатия на "Default Security List for vcn-20221009-1257" (в вашем случае название может отличаться), мы выбираем вкладку "Правила безопасности" и нажимаем на кнопку "Добавить правила для входящего трафика".

У нас появится окно добавления параметров, куда мы вводим данные, как показано у меня на скриншоте.

После всех этих действий нажимаете кнопку "Добавить правила входящего трафика", и мы почти закончили с настройкой и установкой сервера.

Настройка веб-панели

Перезагрузка

sudo reboot

Откройте URL панели и войдите.

Установка ПО (LNMP)

  • Nginx 1.24
  • MySQL MariaDB 10.11
  • PHP 8.3
  • Pure-Ftpd 1.0.49
  • phpMyAdmin 5.2

"Quick install" → "One-click" (до 30 минут)

Python

  1. "App Store" → "Python Manager" → Install версия 2.0
  2. Settings → Version → Install Python 3.10.0

Создание папки

Files → New Folder → testCDN

Путь: /www/wwwroot/testCDN

Развертывание CDN

Загрузка скрипта

https://github.com/illussioon/Fast-Server/tree/main

Распакуйте и загрузите файлы в /www/wwwroot/testCDN

Python-проект

Python Manager → Create project:

  • Name: tutorCDN
  • Path: /www/wwwroot/testCDN/
  • Startup file: /www/wwwroot/testCDN/server.py
  • Port: 8000
  • Run user: Root
  • Startup mode: python

Python-модули

Project → Module → Add:

  • fastapi
  • uvicorn
  • edge-tts
  • requests
  • aiohttp
  • pymysql

Version оставляем пустым.

Плагин GitHub

  1. Files → testCDN → New Folder → plugin
  2. Скачать: GitHub.rar
  3. Распаковать → загрузить папку GitHub в plugin

Настройка GitHub

Personal Access Token

  1. GitHub → Avatar → Settings
  2. Developer Settings → Personal access tokens → Tokens (classic)
  3. Generate new token → Generate new token (classic)
  4. Установите все права, срок действия — без ограничений
  5. Generate token
Важно: Скопируйте токен сразу — он больше не будет показан.

Создание репозитория

  1. GitHub → "New"
  2. Имя репозитория — любое, без пробелов
  3. ПРИВАТНЫЙ репозиторий (не публичный)
  4. "Add README" → ON

Конфигурация

Откройте plugin/GitHub/github_repo.json и заполните:

  • token — ваш GitHub Personal Access Token
  • username — GitHub username
  • repository — название репозитория

Логика работы

Структура: username/repository

Файл в корне Git → youdomain.com/git/filename.zip

В ссылке повторяется полная структура директорий Git.

Настройка домена

Добавление домена на сервер

Python Manager → Project → "Mapping" → youdomain.com

Регистрация домена (nic.ua)

  1. nic.ua → регистрация
  2. Зона: pp.ua - free domain for people!
  3. Нужен загранпаспорт или карта (ByBit/другая)
  4. Привязка карты через Google Pay (до 24 часов)
  5. Верификация домена через Telegram-бота

Cloudflare

  1. Создать FREE аккаунт Cloudflare
  2. Добавить домен → копировать NS-серверы
  3. nic.ua → настройки → NS-серверы → "Изменить" → вставить NS от Cloudflare

Кеширование

Cloudflare → Rules → Page Rules → добавить правило для youdomain.com/cache/git/*

Готово

CDN-система настроена. Файлы транслируются из GitHub репозиториев через ваш сервер.
