## Принципы разработки

1. **Поэтапная разработка** — каждый блок завершается и тестируется отдельно
2. **Модульность** — компоненты и эффекты изолированы по файлам
3. **Прозрачная отладка** — ключевые действия логируются в консоль
4. **Fallback-подход** — визуальные эффекты имеют безопасные значения по умолчанию
5. **Версионирование** — устаревшие материалы переносятся в `archive/`
6. **Чистота и архив** — то, что не используется в продакшене (тяжёлые анимации, старые концепты), складываем в `archive/` вместо удаления.

## Веб-редактор статей (ЛОКАЛЬНО)

Полноценный веб-редактор для написания статей прямо из браузера. **Работает только локально!**

**Особенности:**
- WYSIWYG редактор с поддержкой Markdown
- Автоматическое сохранение в GitHub
- Загрузка изображений через drag & drop
- Автоматическая сборка HTML через GitHub Actions
- Статьи автоматически публикуются на прод

**Использование:**
1. Откройте `admin.html` в браузере (локально, из папки `source/`)
2. Введите GitHub Personal Access Token
3. Начните писать статьи!
4. Статьи автоматически сохраняются в GitHub и собираются в HTML

**⚠️ Важно:** Редактор работает только локально. Не открывайте `admin.html` на проде!

Подробная инструкция:
- [Настройка локального редактора](docs/LOCAL_EDITOR_SETUP.md) - полная инструкция по настройке
- [Инструкция по веб-редактору](docs/ADMIN_EDITOR.md) - как использовать редактор

## Рабочий процесс создания статей

### Вариант 1: Веб-редактор (рекомендуется)
1. Откройте `admin.html`
2. Создайте или откройте статью
3. Напишите контент в редакторе
4. Сохраните в GitHub
5. GitHub Actions автоматически соберёт HTML

### Вариант 2: CLI утилиты
1. `node scripts/add-article.js` - создать статью
2. Отредактировать `blog/content/<id>.md`
3. `node scripts/build-article.js <id>` - собрать HTML
4. Коммит и пуш в GitHub
